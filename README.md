# tailor-resume

応募先の会社・職種に合わせて、履歴書と職務経歴書を自動で最適化する Claude Code カスタムスキルです。

---

## 概要

このスキルは以下を一気通貫で行います。

1. 応募先の会社・求人情報をウェブから収集
2. Google Drive に保存された現在の履歴書・職務経歴書を読み込み
3. 応募先の要件と経歴のギャップを分析
4. **本人の文体・語彙に合わせた**修正案を生成（AI っぽい文体にならないよう PDCA でチェック）
5. 合格するまで自動でレビュー・修正を繰り返す
6. 最終稿を会社ごとのフォルダに自動保存

---

## 前提条件

- [Claude Code](https://claude.ai/code) がインストール済みであること
- Google Drive MCP が Claude Code に接続済みであること
  - 設定方法: Claude Code の MCP 設定から Google Drive を追加する
- Google Drive の `マイドライブ/経歴/` フォルダに以下が保存されていること
  - `履歴書_〇〇.xlsx`（または `.gdoc`）
  - `職務経歴書_〇〇.xlsx`（または `.gdoc`）

---

## セットアップ

### 1. リポジトリをクローン

```bash
git clone https://github.com/ntochio/tailor-resume.git
cd tailor-resume
```

### 2. Claude Code のプロジェクトとして開く

このリポジトリを Claude Code で開くと、`.claude/commands/tailor-resume.md` が自動でカスタムスキルとして認識されます。

または、既存のプロジェクトで使いたい場合は `.claude/commands/` フォルダごとコピーしてください。

```
your-project/
  └── .claude/
        └── commands/
              └── tailor-resume.md
```

### 3. Google Drive の準備

以下のフォルダ構成で履歴書・職務経歴書を配置してください。

```
マイドライブ/
  └── 経歴/
        ├── 履歴書_〇〇.xlsx
        └── 職務経歴書_〇〇.xlsx
```

ファイル名は自由ですが、「履歴書」「職務経歴書」という文字列を含めてください（スキルがキーワードで検索します）。

---

## 使い方

Claude Code のチャットで以下のように入力します。

```
/tailor-resume <会社名> [会社URL] <職種名> [職種URL]
```

### 例

```
/tailor-resume 株式会社〇〇 https://example.com バックエンドエンジニア https://example.com/jobs/123
```

URL なしでも動作します。

```
/tailor-resume 株式会社〇〇 / バックエンドエンジニア
```

---

## 実行フロー

```
入力解析
  ↓
会社・求人情報の収集（URL取得 or Web検索）
  ↓
履歴書・職務経歴書の読み込み（Google Drive）
  ↓
本人の文体プロファイルを抽出
  ↓
ギャップ分析・最適化方針をユーザーに提示・確認
  ↓
初稿作成
  ↓
┌─ 本人らしさチェック ─┐
│ AI臭い表現、難語、    │← 不合格なら修正して再チェック
│ テンプレ表現 etc.     │
└──────────────────────┘
  ↓ 合格
┌─ プロレビュー ───────┐
│ 一貫性、具体性、      │← 不合格なら修正して本人チェックに戻る
│ 要件対応、差別化 etc. │
└──────────────────────┘
  ↓ 全項目合格
最終稿をユーザーに出力 + Google Drive に自動保存
```

---

## 保存先

最終稿は Google Drive に自動で保存されます。会社が同じなら同じフォルダにまとめられ、複数回応募した履歴も日付付きで蓄積されます。

---

## 注意事項

- 元の履歴書・職務経歴書（マスターファイル）は上書きされません
- 実際の経験の範囲内で最適化します。事実にない経歴は追加しません
- 個人情報ファイル（`.xlsx`, `.gdoc` 等）は `.gitignore` で除外済みです
